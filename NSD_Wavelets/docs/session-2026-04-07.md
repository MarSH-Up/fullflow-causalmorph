## 1. Gatekeeper v1-F: Multichannel Consistency Gate

### What was added (`src/detectors/detectors_wavelets.py`)

A new filtering stage (STEP 6c) between step validation and event building:

**`filter_peaks_by_channel_consistency()`**
- For each candidate peak τ, computes per-channel signed energy step: `Δ_ch = median(post) - median(pre)`
- Normalizes by per-channel MAD: `z_ch = Δ_ch / σ_ch` (using existing `_robust_scale()`)
- **Gate**: keeps peak only if `n_active_channels ≥ k_channels_min`
- Also computes **concentration ratio** `R = max|Δ_ch| / Σ|Δ_ch|` (R≈1 = single-channel artifact, R≈1/N = distributed real change)

**`Event` dataclass extended** with:
- `n_active_channels: Optional[int]`
- `frac_active_channels: Optional[float]`
- `channel_z_scores: Optional[np.ndarray]`
- `concentration_ratio: Optional[float]`

**New parameters added to both `detect_nonstationarity()` and `detect_nonstationarity_multimoment()`:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `k_channels_min` | 2 | Min channels that must agree (auto-disabled for N=1) |
| `delta_ch_k` | 1.5 | Per-channel z-score threshold |
| `step_delta_k` | 2.5 | Aggregate step validation threshold (exposed from hardcoded) |

**Pipeline position:**
```
5.  Peak extraction
6a. Step validation on aggregate E_signed     ← existing
6b. Sequential step filter (optional)         ← existing
6c. K-of-channels gate on E_ch               ← NEW (v1-F)
7.  SNR filter + event building               ← existing
```

## 2. Determinism Fixes (`src/scenarios/NonStationaryCausalScenarios.py`)

### Problem
Results were non-deterministic across process runs. Same seed, different output.

### Root cause
`list(set_A - set_B)` in `calculate_structural_change()` iterates in PYTHONHASHSEED-dependent order. Different processes → different DAG structures → different time series.

### Fix: sorted edge lists
```python
# Before (non-deterministic):
edges_can_add = list(all_possible_edges - base_edges)
edges_can_remove = list(base_edges)

# After (deterministic):
edges_can_add = sorted(all_possible_edges - base_edges)
edges_can_remove = sorted(base_edges)
```

### Secondary fix: Apple Accelerate BLAS spurious FP exceptions
`np.zeros @ weights` triggered spurious divide-by-zero warnings via Apple Accelerate BLAS, which corrupted the FP status register and caused rounding-level non-determinism downstream.

Fixes applied:
1. DataFrame initialized to zeros (not NaN): `np.zeros((nsamples, p))`
2. `@` operator replaced with `np.dot(np.ascontiguousarray(...), weights)`
3. Node values clipped: `np.clip(parent_effect + error, -50, 50)`

### Secondary fix: BLAS thread pinning
Added at top of `run_experiment.py` (before numpy import):
```python
os.environ.setdefault('OMP_NUM_THREADS', '1')
os.environ.setdefault('OPENBLAS_NUM_THREADS', '1')
os.environ.setdefault('MKL_NUM_THREADS', '1')
os.environ.setdefault('VECLIB_MAXIMUM_THREADS', '1')
os.environ.setdefault('NUMEXPR_NUM_THREADS', '1')
```

**Outcome**: 5/5 consecutive runs produce identical results.

### Onset vs. offset clarification
Regime transitions where variance **decreases** (e.g., high-variance → low-variance) produce negative dE peaks → correctly detected as **offsets**, not onsets. Both onsets and offsets are valid change points and are evaluated together via `result.change_points`.

### Duplicate detection fix

## 4. Tolerance Shading in Plots

Added `±N` shaded bands around each ground-truth change point in `plot_multimoment_results()`:

```python
def _draw_gt_bands(ax):
    for i, cp in enumerate(ground_truth):
        if tolerance > 0:
            ax.axvspan(cp - tolerance, cp + tolerance,
                       alpha=0.18, color='#2ecc71', zorder=0,
                       label=f'±{tolerance} window' if i == 0 else '')
        ax.axvline(cp, **gt_style, label='Ground Truth' if i == 0 else '', zorder=10)
```

`plot_multimoment_results()` now accepts `tolerance: int = 0` parameter. All three test calls pass `tolerance=125`.
