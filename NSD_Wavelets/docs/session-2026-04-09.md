## 1. Near-Duplicate Detection Merging (`detectors_wavelets.py`)

### Problem
Two detections close in time (e.g. an onset and offset from the same structural change) both survived as separate change points.

### Fix: `_merge_nearby_change_points()`
Added private helper after `_robust_scale`:

```python
def _merge_nearby_change_points(change_points: List[int], min_distance: int = 200) -> List[int]:
    sorted_cps = sorted(change_points)
    merged = [sorted_cps[0]]
    for cp in sorted_cps[1:]:
        if cp - merged[-1] < min_distance:
            merged[-1] = (merged[-1] + cp) // 2   # midpoint
        else:
            merged.append(cp)
    return merged
```

Applied at both assembly points in the detector:
- `detect_nonstationarity`: replaces `sorted(set(onset_points + offset_points))`
- `detect_nonstationarity_multimoment`: same replacement

Threshold: 200 samples (exclusive). Distinct from the `simplify_transitions(min_distance=160)` evaluation pass that also runs downstream.

## 2. SHD Computation Fix (`full_pipeline.py`)

### Problem
`compute_shd()` delegated to `normalized_shd()` which only returned `{"shd": ..., "normalized_shd": ...}`. The rest of the pipeline read `"SHD"` (wrong capitalisation), `"F1"`, `"Precision"`, `"Recall"` — all `KeyError` at runtime.

### Fix
`compute_shd()` is now self-contained:

```python
def compute_shd(pred_adj, true_adj, threshold=0.05) -> dict:
    gt_bin  = (gt != 0)
    est_bin = (np.abs(est) > threshold)
    # remove diagonal
    tp = int(np.sum(gt_bin & est_bin))
    fp = int(np.sum(~gt_bin & est_bin))
    fn = int(np.sum(gt_bin & ~est_bin))
    shd = fp + fn
    return {"SHD": shd, "normalized_shd": ..., "F1": ..., "Precision": ..., "Recall": ...}
```

Removed the now-unused `from utils.metrics import normalized_shd` import.

## 3. Brain-Inspired Learning Trajectory (`full_pipeline.py`)

### Concept
Previous versions used either (a) a single fixed graph for all regimes or (b) random independent structural changes. The new model: nodes are fixed brain regions, edges rewire from a sparse initial state (`G_initial`) toward a denser final target (`G_target`).

### Implementation in `build_nonstationary_scenario()`

**Graph generation:**
```python
G_initial  # p-conn × 0.6  (sparse start)
G_target   # p-conn × 1.4  (denser learned state)
```

**Edge partitioning:**
```
stable_edges  = G_initial ∩ G_target  (always present)
edges_to_lose = G_initial \ G_target  (removed over time)
edges_to_gain = G_target  \ G_initial (added over time)
```

**Scheduling formula** — avoids the `round()` collapse that made regime 0 and 1 identical:
```python
def _schedule(edges):
    n = len(edges)
    return {e: 1 + int(k * n_trans / n) for k, e in enumerate(edges)}
```

Change $k$ fires at regime $1 + \lfloor k \cdot (R-1) / n \rfloor$. This guarantees the first change always fires at regime 1, so $G_0 \neq G_1$ for any $n \geq 1$.

**Per-regime edge set:**
```python
current_edges = (
    stable_edges
    | {e for e, r in lose_at.items() if i < r}   # not yet removed
    | {e for e, r in gain_at.items() if i >= r}  # already gained
)
```

**`change_infos`** now records structural changes (edges added/removed) per transition instead of the previous parametric deltas.

**Print output** now shows:
```
Edges per regime   : [2, 3, 3, 4, 5]  (trajectory: 2 → 5)
Regime 0 → 1: +1 edge(s) [V1→V3]  -0 edge(s) [—]
```

## 4. Bayesian Structure Aggregation (`full_pipeline.py`)

### Model
Beta-Bernoulli aggregation over all regime structures, weighted by regime length.

For each edge $(i, j)$:
- **Prior**: $\text{Beta}(1, 1)$ (uniform)
- **Weighted vote**: $\bar{x}_{ij} = \sum_r w_r \hat{A}^{(r)}_{ij}$ where $w_r = n_r / \sum n_r$
- **Posterior mean**: $\hat\theta_{ij} = (1 + R\bar{x}_{ij}) / (2 + R)$
- **Decision**: include edge if $\hat\theta_{ij} > 0.20$

Threshold changed from 0.50 → **0.20** to compensate for trajectory-induced dilution: early regimes lack edges that only appear in `G_target`, so a permissive threshold is needed to recover the final structure.

### Plot: `plot_consensus_structure()` — 3 panels
1. **Left**: Bayesian consensus graph, edges coloured by posterior probability (Blues colormap)
2. **Middle**: Global ground truth (`true_adjs[-1]` = final target $G_\text{tgt}$), with nSHD + F1 vs. consensus in title
3. **Right**: Posterior probability heatmap; **green border** on cells where the GT has an edge (immediate TP/FP/FN visualisation)

`true_adjs` added to `run_full_pipeline` return dict so it's accessible downstream.

## 5. Batch Experiment Runner (`batch_experiments.py`)

New standalone file for sweeping over node counts and running N independent experiments.

### Usage
```bash
python batch_experiments.py --p_min 3 --p_max 7 --n_experiments 10
python batch_experiments.py --p_min 3 --p_max 5 --n_experiments 3 --output results/sweep.csv
python batch_experiments.py --verbose   # show per-run output
```

### CSV columns
| Group | Columns |
|-------|---------|
| ID | `exp_id`, `seed`, `p`, `n_regimes`, `total_samples`, `n_true_cps`, `n_detected_cps` |
| Detection | `det_f1`, `det_precision`, `det_recall`, `det_tp`, `det_fp`, `det_fn` |
| Structure (avg) | `mean_shd`, `mean_norm_shd`, `mean_struct_f1`, `mean_struct_precision`, `mean_struct_recall`, `n_structures` |
| Consensus vs GT | `consensus_shd`, `consensus_norm_shd`, `consensus_f1`, `consensus_precision`, `consensus_recall` |
| Status | `status` (`ok`/`error`), `error_msg` |

- Failed runs write an error row instead of crashing the batch.
- Output directory is created automatically (`os.makedirs(..., exist_ok=True)`).
- Per-$p$ summary (mean ± std of all key metrics) printed inline.
- Global summary printed at end.

### First batch results (p=3..7, 5 runs each = 25 experiments)
```
det_f1             mean=0.727  std=0.199
det_precision      mean=0.865  std=0.192
det_recall         mean=0.670  std=0.247
mean_norm_shd      mean=0.208  std=0.093
mean_struct_f1     mean=0.484  std=0.220
consensus_norm_shd mean=0.264  std=0.110
consensus_f1       mean=0.342  std=0.289
```

**Assessment:** Detection precision is solid (0.865); recall (0.670) and structural F1 (0.484) are mediocre. Consensus F1 (0.342) is worse than per-regime F1 due to the trajectory dilution problem — partially addressed by the 0.20 threshold but not yet fully resolved.

## Pending / Known Issues

- **Consensus dilution**: The 0.20 threshold partially compensates but the fundamental issue remains — the Bayesian prior should ideally increase over the trajectory (later regimes should have higher weight), not just length-weighted. Consider a **recency-weighted** scheme where $w_r \propto r$ (or $w_r \propto r \cdot n_r$).
- **Structural F1 ≈ 0.48**: Near coin-flip. Likely causes: (a) detected windows don't align well with true regime boundaries; (b) CausalMorph warm-start may be hurt by accumulated errors in the prior chain when earlier regimes have different topology.
- **Detection recall ≈ 0.67**: One in three change points is missed. Consider lowering `step_delta_k` further or tuning `min_snr`.
